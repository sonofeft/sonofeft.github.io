switch tag { }
In example below "tag" is equal to "5"
Each case value MUST be unique
NOTE: compiler will flag different cases that are equal
switch i := a + 2; i { }
switch { }
Each case is a boolean statement
NOTE: compiler will not catch overlapping case boolean
No "break" statement required like C or C++
HOWEVER, can "fall through" to next statement (regardless of next tag)
var i interface{} = 1 // an interface variable of type int
switch i.(type) { }
i.e. "case int" or "case float64" etc.
break out of case in switch with "break" statement
For loop is only loop structure
for i:=0; i<5; i++ { }
for i:=0; i<5; i = i + 2 { }
Can "screw" with i inside loop... not recommended.
"i" is scoped to for loop, not containing function
Doesn't exist in Go, however
for i < 5 { i++ } // acts like while i < 5
Undetermined exit... use "break"
e.g. if i==5 { break }
Jumps to end of loop and
NOTE: A "break" statement in "j" loop will only break out of the j logic.
Need a LABEL to break out of both i and j from inside the j loop
for key, value := range s { }
Works for slices, arrays, maps, strings and CHANNELS
NOTE: key,value for string is int, int (can convert unicode back with "string(val)")
NOTE: may need to say "for _,v in range s" if key not needed
simple loops
exiting early
looping over collections
defers call until about to exit
NOTE: Last In First Out
Good for closing resources. Can place "defer close" right after "open"
NOTE: Evaluates defer statement at the time it is executed
When application can't figure out what to do
Call "panic" function to raise a fatal exception
panic( "panic output string" )
NOTE: for err flags, can panic with "err.Error()"
any defer statements will execute after a panic.
SO... panic statements are executed AFTER defer statements
NOTE: defer takes a function CALL... NOT just a function (i.e. func() )
A panic in a function with a "defer func() { ...recover() }" will not bring down application
i.e. after main calls panicker, main will continue even though panicker panics.


Using & and * to reference and dereference variable
Use %p to print a pointer with Printf
NOTE: integer array locations are 4 bytes apart
NOTE: Go does NOT allow pointer arithmetic (e.g. &a + 4 ) "unsafe" package can however
a = new( myStruct ) // a is a pointer to a myStruct with all zero values
NOTE: pointer variable is "nil" until "new" is called
functions that accept pointer inputs need to check for "nil"
Dereferencing struct simply requires dot notation: a.name
b := a // for array makes 2nd copy of array
b := a // for slice makes 2nd copy of pointer to underlying array
NOTE: maps behave like slices (Again they are pointers to underlying data)
NOTE: structs behave like arrays (Make 2nd full copy of struct data)
