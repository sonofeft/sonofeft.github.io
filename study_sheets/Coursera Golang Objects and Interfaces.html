Coursera Golang Objects and Interfaces
Identical at high level, Different at low level
e.g. Rectangle and Triangle both have an Area method (i.e. polymorphic)
NOTE: GoLang does NOT have Inheritance or Overriding of parent objects
Helps to achieve polymorphism
set of method signatures (name, parameters, return types)

uses "interface" keyword
NOTE: No need to specify that Triangle has interface Shape2D... compiler figures it out
The interface eventually gets mapped to a concrete value
Can make a variable of interface type (e.g. var s1 Speaker)
Note: the assignment "s1 = d1" is legal because Dog satisfies Speaker interface
Note: Dynamic type of s1 is then "Dog" and its dynamic value is "d1".
Even w/o data, can find Dog Speak method
NOTE: needed to handle "nil" case within Speak
Can't call any methods... i.e. only know that s1 is a Speaker, but no associated method
e.g. function takes multiple types (int, float, string)
example FitInYard() takes any shape to see if pool fits in yard ( Area and Perimeter interface)
FitInYard takes any type that satisfies Shape2D interface
No methods specified, e.g. PrintMe( val interface{} )
Takes any variable as input... no defined methods required.
use Type Assertions
rect, ok := s.(Rectangle) --> returns concrete rect, true OR zero, false
