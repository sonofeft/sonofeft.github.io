create goroutine: "go funcname()" // creates a green thread
NOTE: main routine is also a goroutine.
Can also use anonymous function as goroutine (see example below)
NOTE: Sleep is bad approach to delay main goroutine from terminating.
A less expensive thread model, that go and erlang use.
cheap to create and destroy... unlike operating system threads.
goroutine scheduler does not interrupt main thread until Sleep command.
Example below prints "Goodbye" for that reason.
i.e. the closure is not evaluated until the main thread is interrupted.
In general DO NOT access local variables from a goroutine.
Create WaitGroup variable: var wg = sync.WaitGroup{} // note curly brackets
For each goroutine call: wg.Add( 1 )
NOTE: in example, wg is local to main but accessed by goroutine
WaitGroup is designed more to be used as a global variable in main
WARNING... example is unsynchronized, so makes a mess.
goroutines are racing against each other.
A lock that the application will honor
i.e. protect some data from change
Can lock for Read and/or Write (e.g. may have many readers, but only 1 writer)
Place Locks OUTSIDE of goroutines... Unlock INSIDE goroutines
Sets max cores/os threads: runtime.GOMAXPROCS( 8 ) // 8 cores in this example
Get number of assigned cores: runtime.GOMAXPROCS( -1 ) // what was set by previous call
Can restrict application to a single core: runtime.GOMAXPROCS( 1 )
Tuning: perhaps runtime.GOMAXPROCS( 100 ) will optimize your application
Use: go run -race
ALLWAYS USE: go run -race OR go build - race
Set runtime.GOMAXPROCS > 1 in development to identify any race conditions.

synch data flow between goroutines
Create channel with "make" function: ch := make( chan int ) // this is an int channel
ch <- 42 // puts 42 on channel (BLOCKS until channel accepts it)
i := <- ch // receives channel data (BLOCKS until data comes through)
Can specify in function parameter list the direction of the channel
NOTE: the call simply uses "(ch)"... the compler handles the directional issues
Specify size of buffer: ch := make( chan int, 50 ) // buffer size = 50
must close channel to let receiver know the end happend: close( ch )
NOTE: once closed, a channel can NOT be reopend (will panic if used after close)
if i, ok := <- ch; ok { } else { }
for goroutine that loops over channel, main can use: defer func() { close(ch) }
OR signal-only channel ( sends and empty struct )
NOTE: An empty struct "struct{}", requires zero memory
goroutine can end when "doneCh" issues and empty struct message.
